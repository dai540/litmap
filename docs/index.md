# litmap

`litmap` is a Python package for reproducible literature mapping.
It is built for research and review workflows that need explicit artifacts,
stable reruns, and a clean separation between clustering logic and map display.

## Why this package exists

Most literature mapping projects start as notebooks and end as a folder full of
half-explained intermediate files. `litmap` takes the opposite approach:

- treat the run directory as a public interface
- keep ingestion, embedding, analysis, and visualization separate
- preserve the difference between analysis space and display space
- make it easy to explain what a cluster means and where it came from

## What `litmap` aims to produce

A completed run should leave behind a human-readable artifact tree:

```text
runs/
  immunotherapy-landscape/
    manifest.json
    config.snapshot.yaml
    raw/
      source_query.txt
      source_records.parquet
    corpus/
      papers.parquet
      text_inputs.csv
      dedup_report.csv
    embeddings/
      embeddings.npy
      embeddings_l2.npy
    analysis/
      pca50.npy
      labels.csv
      cluster_summary.csv
      coords_2d.csv
    reports/
      map_interactive.html
```

## Pipeline at a glance

The easiest way to understand `litmap` is to think in five stages:

1. define the corpus
2. prepare normalized paper records
3. build embeddings
4. cluster in analysis space
5. project into a separate 2D display space

That split between **analysis space** and **display space** is the most important
idea in the package. Cluster membership is decided in the higher-dimensional
analysis pipeline, while the visible map is only a readable projection.
