Tutorials

Case-study articles

Worked examples that show what one `litmap` run looks like when read as a review artifact, not just as code execution.

Each article starts with a domain question, describes the corpus and analysis setup, then walks through cluster structure, PCA diagnostics, artifact tables, and review-oriented interpretation. The goal is to make the output usable for triage, reading order, and follow-up decisions.

Shared structure

What each article contains

  • Headline findings
    The fastest summary of what structure the run exposed.
  • PCA diagnostics
    Tables and plots that explain the visible map geometry.
  • Cluster interpretation
    Topic-specific reading of dense neighborhoods and bridge regions.
  • Review operations
    Reading order, owner assignment, and search expansion ideas.
Artifacts

Files behind each article

  • labels.csv
    Paper-level labels and cluster membership strength.
  • cluster_summary.csv
    Cluster size, persistence, and article-facing theme summaries.
  • coords_2d.csv
    2D plotting coordinates for the display map.
  • map_interactive.html
    Interactive Plotly view linked from the article.
